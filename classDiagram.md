# Class Diagram 

This class diagram represents the structural design of the FlexiPay system.

It illustrates:

- Layered architecture (Controller → Service → Repository)
- Domain entities and their relationships
- Multi-tenant modeling
- Strategy Pattern for payment processing
- State Pattern for subscription lifecycle management
- Object-oriented principles such as abstraction, encapsulation, and polymorphism
